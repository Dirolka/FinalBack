# Product Manager

Веб-приложение для управления товарами и категориями с авторизацией (JWT), ролевым доступом (RBAC) и оформлением заказов.

---

## Стек

| Часть       | Технологии                    |
|------------|-------------------------------|
| Backend    | Node.js, Express, MongoDB (Mongoose) |
| Авторизация| JWT, bcrypt                   |
| Frontend   | HTML, CSS, JavaScript (SPA с hash-маршрутизацией) |

---

## Требования и установка

- **Node.js** 18+
- **MongoDB** (локально или Atlas)

```bash
git clone <repo-url>
cd final
npm install
```

Создайте файл `.env` в корне:

```env
PORT=3000
MONGODB_URI=mongodb://127.0.0.1:27017/assignment4
JWT_SECRET=ваш_секретный_ключ
JWT_EXPIRES_IN=7d
```

```bash
npm run dev
```

Приложение: **http://localhost:3000**

## Схема БД (MongoDB)

База данных: **assignment4** (или значение из `MONGODB_URI`).

### Коллекция `users`

| Поле       | Тип     | Обязательное | Описание |
|------------|---------|--------------|----------|
| `_id`      | ObjectId| да           | Идентификатор |
| `email`    | String  | да, уникальный | Email (lowercase) |
| `password` | String  | да           | Хеш bcrypt (не возвращается в запросах по умолчанию) |
| `role`     | String  | да           | `"user"` \| `"admin"` |
| `createdAt`| Date    | да           | Время создания |
| `updatedAt`| Date    | да           | Время обновления |

---

### Коллекция `categories`

| Поле        | Тип     | Обязательное | Описание |
|-------------|---------|--------------|----------|
| `_id`       | ObjectId| да           | Идентификатор |
| `name`      | String  | да           | Название (2–100 символов) |
| `model`     | String  | нет          | Модель (до 100 символов) |
| `memory`    | String  | нет          | Объём памяти (до 50 символов) |
| `color`     | String  | нет          | Цвет (до 50 символов) |
| `description` | String | нет        | Описание (до 500 символов) |
| `createdAt` | Date   | да           | Время создания |
| `updatedAt` | Date   | да           | Время обновления |

---

### Коллекция `products`

| Поле         | Тип       | Обязательное | Описание |
|--------------|------------|--------------|----------|
| `_id`        | ObjectId   | да           | Идентификатор |
| `name`       | String     | да           | Название (минимум 2 символа) |

| `category`   | ObjectId   | да           | Ссылка на `categories._id` |

### Коллекция `orders`

### Auth

#### POST `/auth/register`

Регистрация пользователя.

**Доступ:** публичный.

**Тело запроса (JSON):**

| Поле      | Тип    | Обязательное | Описание |
|-----------|--------|--------------|----------|
| `email`   | string | да           | Email |
| `password`| string | да          | Пароль (минимум 6 символов) |
| `role`    | string | нет          | `"user"` \| `"admin"` (по умолчанию `"user"`) |

**Ответ (201):**

```json
{
  "message": "User registered successfully",
  "user": { "id": "<_id>", "email": "...", "role": "user" },
  "token": "<JWT>"
}
```

**Ошибки:** `400` (нет email/password), `409` (email уже занят).

---

#### POST `/auth/login`

Вход в систему.

**Доступ:** публичный.

**Тело запроса (JSON):**

| Поле       | Тип    | Обязательное |
|------------|--------|--------------|
| `email`    | string | да           |
| `password` | string | да           |

**Ответ (200):**

```json
{
  "message": "Login successful",
  "user": { "id": "<_id>", "email": "...", "role": "..." },
  "token": "<JWT>"
}
```

**Ошибки:** `400` (нет полей), `401` (неверный email или пароль).

---

#### GET `/auth/me`

Текущий пользователь по JWT.

**Доступ:** требуется JWT.

**Ответ (200):**

```json
{
  "user": {
    "id": "<_id>",
    "email": "...",
    "role": "...",
    "createdAt": "<ISO date>"
  }
}
```

**Ошибки:** `401` (нет или неверный токен).

---

### Products

#### GET `/products`

Список товаров с опциональной фильтрацией.

**Доступ:** публичный.

**Query-параметры:**

| Параметр   | Тип    | Описание |
|------------|--------|----------|
| `category` | string | ID категории |
| `model`    | string | Модель (из категории) |
| `color`    | string | Цвет (из категории) |
| `memory`   | string | Память (из категории) |
| `minPrice` | number | Минимальная цена |
| `maxPrice` | number | Максимальная цена |

**Ответ (200):**

```json
[
  {
    "_id": "...",
    "name": "...",
    "price": 100,
    "category": { "_id": "...", "name": "...", "model": "...", "memory": "...", "color": "..." },
    "description": "...",
    "createdAt": "...",
    "updatedAt": "..."
  }
]
```

---

#### GET `/products/:id`

Один товар по ID.

**Доступ:** публичный.

**Ответ (200):** объект товара с заполненным `category`.

**Ошибки:** `400` (некорректный id), `404` (товар не найден).

---

#### POST `/products`

Создание товара.

**Доступ:** JWT + роль `admin`.

**Тело запроса (JSON):**

| Поле         | Тип    | Обязательное |
|--------------|--------|--------------|
| `name`       | string | да (≥ 2 символа) |
| `price`      | number | да (≥ 0) |
| `category`   | string | да (ObjectId категории) |
| `description`| string | да (≥ 5 символов) |

**Ответ (201):** созданный товар с заполненным `category`.

**Ошибки:** `400` (валидация, категория не найдена), `401`/`403` (нет доступа).

---

#### PUT `/products/:id`

Обновление товара.

**Доступ:** JWT + роль `admin`.

**Тело запроса (JSON):** те же поля, что и у POST (все обязательные для обновления).

**Ответ (200):** обновлённый товар.

**Ошибки:** `400`, `404`, `401`/`403`.

---

#### DELETE `/products/:id`

Удаление товара.

**Доступ:** JWT + роль `admin`.

**Ответ (200):**

```json
{ "message": "Product deleted", "id": "..." }
```

**Ошибки:** `400`, `404`, `401`/`403`.

### Orders

#### POST `/orders`

Создание заказа (покупка).

**Доступ:** JWT (любая роль).

**Тело запроса (JSON):**

| Поле      | Тип    | Обязательное | Описание |
|-----------|--------|--------------|----------|
| `product` | string | да           | ID товара |
| `quantity`| number | нет          | Количество (по умолчанию 1, минимум 1) |

**Ответ (201):** созданный заказ с заполненными `product` и `user`:

```json
{
  "_id": "...",
  "user": { ... },
  "product": { "_id": "...", "name": "...", "price": ... },
  "quantity": 1,
  "totalPrice": 100,
  "status": "pending",
  "createdAt": "...",
  "updatedAt": "..."
}
```

**Ошибки:** `400` (нет product, некорректный id), `404` (товар не найден), `401`.

---

#### GET `/orders`

Список заказов текущего пользователя.

**Доступ:** JWT.

**Ответ (200):**

```json
[
  {
    "_id": "...",
    "product": { "_id": "...", "name": "...", "price": ..., "category": ... },
    "quantity": 1,
    "totalPrice": 100,
    "status": "pending",
    "createdAt": "...",
    "updatedAt": "..."
  }
]
```

**Ошибки:** `401`.

## Роли и доступ

| Действие              | Публичный | user | admin |
|-----------------------|-----------|------|-------|
| GET товары/категории  | да        | да   | да    |
| Регистрация / Вход    | да        | да   | да    |
| POST/PUT/DELETE товары/категории | нет | нет | да    |
| POST/GET заказы       | нет       | да   | да    |
