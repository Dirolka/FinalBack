# SLI, SLO и Error Budget (для отчёта)

## Примеры SLI

| SLI            | Определение                                      |
|----------------|--------------------------------------------------|
| Availability   | Доля запросов с кодом ответа \< 500              |
| Latency        | Доля запросов быстрее 500 мс                     |

## SLO и Error Budget (месяц ≈ 30 дней)

### Availability SLO = 99,5%

- Минут в месяце: \(30 \times 24 \times 60 = 43\,200\) мин.
- Error budget по даунтайму: \((1 - 0{,}995) \times 43\,200 = 216\) минут допустимого простоя за месяц.

### Latency SLO = 95% запросов быстрее 500 мс

- Если в день 10 000 запросов → в месяц \(10\,000 \times 30 = 300\,000\) запросов.
- Допустимая доля «медленных» (не попавших в цель): 5%.
- Error budget: \(0{,}05 \times 300\,000 = 15\,000\) запросов с задержкой ≥ 500 мс за месяц.

*(Цифры запросов в день подставь свои из реальной нагрузки или оценки.)*

## Как проверить алерт ServiceDown вручную

```bash
docker compose stop backend
```

Через ~30 секунд в Prometheus → **Alerts** статус **FIRING** для `ServiceDown`. Сделай скриншот, затем:

```bash
docker compose start backend
```
